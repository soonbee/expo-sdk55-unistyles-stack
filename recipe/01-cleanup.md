# 01. Package and code cleanup

Audience: AI agents reproducing this setup from a fresh Expo template.

## Context

- Starting point: project generated by `npx create-expo-app@latest --template default@sdk-55`
- Goal: strip the template down to a single screen that renders one `<Text>`, while keeping commonly-used packages that are likely to be reinstalled later.

## Guiding principle

When deciding whether to drop a dependency, apply both filters:

1. **Not referenced** by the minimal single-screen app.
2. **Not commonly reinstalled** in typical Expo projects.

Drop only when both are true. Keep a package if it is a widely-used Expo/RN staple (fonts, image, safe-area, reanimated, gestures, screens, router peers, etc.), even if the current minimal screen does not import it.

## Step 1 — Edit `package.json`

Remove these dependencies and their reasons:

| Package                         | Reason                                                         |
| ------------------------------- | -------------------------------------------------------------- |
| `@react-navigation/bottom-tabs` | Template-only tab navigation; unused with single screen        |
| `@react-navigation/elements`    | Paired with bottom-tabs for default header/tab UI              |
| `expo-device`                   | Device info utility only used by template welcome screen       |
| `expo-glass-effect`             | iOS 26 liquid-glass effect, niche                              |
| `expo-symbols`                  | Apple SF Symbols, niche                                        |
| `expo-web-browser`              | In-app browser; cheap to reinstall when needed                 |
| `react-dom`                     | Web React renderer; this project never ships web (see step 02) |
| `react-native-web`              | DOM adapter for RN primitives; ditto                           |

Keep everything else, including: `@react-navigation/native`, `expo`, `expo-constants`, `expo-font`, `expo-image`, `expo-linking`, `expo-router`, `expo-splash-screen`, `expo-status-bar`, `expo-system-ui`, `react`, `react-native`, `react-native-gesture-handler`, `react-native-worklets`, `react-native-reanimated`, `react-native-safe-area-context`, `react-native-screens`, `@types/react`, `typescript`.

Also remove the `reset-project` script (the script file is deleted in Step 4).

## Step 2 — Simplify `src/app`

Replace `src/app/_layout.tsx` with a minimal Stack (no theme provider, no splash overlay, no custom tabs):

```tsx
import { Stack } from 'expo-router';

export default function RootLayout() {
  return <Stack screenOptions={{ headerShown: false }} />;
}
```

Replace `src/app/index.tsx` with a single-`Text` screen:

```tsx
import { StyleSheet, Text, View } from 'react-native';

export default function HomeScreen() {
  return (
    <View style={styles.container}>
      <Text>Hello, Expo!</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    alignItems: 'center',
    justifyContent: 'center',
  },
});
```

Delete the extra tab screen:

```sh
rm src/app/explore.tsx
```

## Step 3 — Delete unused `src/` subdirectories

All of these exist only to support the template's welcome UI.

```sh
rm -rf src/components src/constants src/hooks src/global.css
```

After this step, `src/` contains only `app/`.

## Step 4 — Clean assets and scripts

Remove template welcome-screen images. Keep icon/splash/adaptive-icon assets because `app.json` still references them.

```sh
rm -rf \
  assets/images/tabIcons \
  assets/images/react-logo.png \
  assets/images/react-logo@2x.png \
  assets/images/react-logo@3x.png \
  assets/images/expo-badge.png \
  assets/images/expo-badge-white.png \
  assets/images/expo-logo.png \
  assets/images/logo-glow.png \
  assets/images/tutorial-web.png \
  assets/images/favicon.png
```

`favicon.png` goes with the web-only deps removed above — step 02 also deletes the web block in `app.json` that was its only consumer.

Keep: `icon.png`, `splash-icon.png`, `android-icon-background.png`, `android-icon-foreground.png`, `android-icon-monochrome.png`, and the `expo.icon/` directory.

Remove the reset-project helper:

```sh
rm -rf scripts
```

`app.json` needs no changes — its references point only to assets that are still present.

## Step 5 — Verify

```sh
npm install
npx tsc --noEmit
```

Expected: `npm install` completes without errors; `tsc --noEmit` exits with no output.

## Notes for future steps

- `tsconfig.json` path aliases `@/*` → `./src/*` and `@/assets/*` → `./assets/*` remain untouched and are safe to rely on later.
- `app.json` enables `experiments.typedRoutes` and `experiments.reactCompiler`; do not disable these when adding later tooling unless a step explicitly requires it.
- `expo-router` is the entry (`"main": "expo-router/entry"`) and requires `@react-navigation/native`, `expo-linking`, `react-native-screens`, `react-native-safe-area-context` as peers — hence their retention.
- Web deps (`react-dom`, `react-native-web`) and `favicon.png` are dropped here because step 02 disables the web target. If a future reproduction needs web, skip those removals and also skip step 02.
